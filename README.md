## Retrieval: ai-forever/FRIDA
Выбор модели обоснован балансом производительности и качества:
SOTA Performance: Топ-3 в категории Retrieval на бенчмарке MTEB (Russian).
Efficiency: Компактный размер (< 1 млрд параметров).
Optimal Context: Окно в 512 токенов полностью покрывает целевой размер чанка.

## Парсер
Разработан специализированный модуль на базе BeautifulSoup4 для извлечения данных из Базы Знаний gu.spb.ru:
**Semantic Chunking:** Разбиение текста на логические блоки «Вопрос-Ответ». Это гарантирует, что каждый чанк содержит законченную мысль, а не случайный набор предложений.
**Context Enrichment:** Добавление метаданных (название раздела) к каждому фрагменту для повышения точности поиска.

## SGR
Модуль генерации построен на базе концепции SGR.
**Chain-of-Verification:**
Extraction: Сначала модель принудительно выписывает ключевые сущности (даты, суммы, ведомства). Это снижает риск галлюцинаций в основном тексте.
Reasoning (Analysis): Модель обосновывает право на услугу, связывая факты с нормами из контекста.
Algorithmic Output: Формирование четкого пошагового плана действий для гражданина.
**Prompt Engineering:**
Внедрен Few-Shot пример (эталон структуры) прямо в системный промпт для стабилизации формата JSON.

## Evaluation
Для объективного контроля качества генерации используется LLMJudge. Пример отчета в RAG_Eval.ipynb.
RAG Triad Metrics: Оценка проводится по трем ключевым осям:
Faithfulness: Проверка на галлюцинации. Ответ должен опираться только на предоставленный контекст.
Relevance: Соответствие ответа интенту пользователя.
Completeness: Полнота извлеченных фактов.
